# Solution Seeking Process

Use this process when turning an open-ended question into a concrete solution.

## 1. Define The Ask

- What result is needed?
- Who will use the result?
- What does a successful answer or implementation look like?

## 2. Capture Context

- Existing files, links, systems, or prior conversations.
- Constraints such as time, tools, platform, data access, cost, or privacy.
- Assumptions that need validation.

## 3. Explore Options

- Candidate approaches.
- Benefits and risks.
- Dependencies or blockers.

## 4. Decide

- Selected approach.
- Reason for choosing it.
- What will be deferred.

## 5. Verify

- Tests, checks, review steps, or manual validation.
- Evidence that the solution handles the original ask.
- Follow-up work if verification is incomplete.
